# 네트워크 설정 가이드

## 우리 현장 네트워크 구성

| 항목 | 값 | 용도 |
|------|----|------|
| 컨트롤 박스 고정 IP | `10.0.2.7` | 외부 제어용 기본 IP |
| 명령 포트 | `5000` | 스크립트 명령 전송 |
| 상태 포트 | `5001` | `reqdata`로 로봇 상태 수신 |
| 통신 방식 | TCP/IP 유선 LAN | PC ↔ 컨트롤 박스 직접 연결 |
| 308호 로봇암 IP | `172.16.3.128` | rbpodo 라이브러리 사용 시 |

> 두 IP가 다른 이유: `10.0.2.7`은 소켓 직접 제어용, `172.16.3.128`은 rbpodo / cobot API 사용 시 환경에 따라 다를 수 있습니다. 실제 사용 환경에서 확인 후 사용하세요.

## PC 네트워크 설정 방법

로봇과 같은 대역으로 PC 유선 LAN을 고정 IP로 설정합니다.

**Windows 기준:**

1. 제어판 → 네트워크 및 공유 센터 → 어댑터 설정 변경
2. 유선 LAN 어댑터 우클릭 → 속성
3. `IPv4` 선택 → 속성
4. 아래와 같이 입력:

       IP 주소:        10.0.2.x   (x는 7 제외 임의값, 예: 10.0.2.10)
       서브넷 마스크:   255.255.255.0
       기본 게이트웨이: (비워두기)

## 연결 테스트

```bash
# 컨트롤 박스에 핑 확인
ping 10.0.2.7

# 포트 열려있는지 확인 (Windows)
telnet 10.0.2.7 5000
```

```python
# Python으로 연결 확인
import socket

ROBOT_IP = "10.0.2.7"
COMMAND_PORT = 5000

try:
    sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    sock.settimeout(3)
    sock.connect((ROBOT_IP, COMMAND_PORT))
    print("연결 성공")
    sock.close()
except Exception as e:
    print(f"연결 실패: {e}")
```

## 절대 주의사항

> **로봇 연결 중 절대 하지 말 것**

- VPN 켜기 → 라우팅 변경으로 로봇 IP 접근 차단
- 네트워크 어댑터 IP 변경
- 다른 네트워크 자동 IP 설정으로 전환
- 포트 5000 / 5001을 다른 프로그램이 사용하도록 방치

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| `Connection refused` | 컨트롤 박스 꺼짐 또는 IP 불일치 | 박스 전원 확인, IP 재확인 |
| `Timeout` | PC IP 대역 불일치 또는 VPN 활성 | PC 네트워크 설정 확인, VPN 종료 |
| 연결됐다가 끊김 | 케이블 불량 또는 네트워크 변경 | 케이블 교체, 고정 IP 재확인 |
| 포트 5000 응답 없음 | 다른 프로그램이 포트 점유 | 작업관리자에서 포트 점유 프로세스 종료 |
